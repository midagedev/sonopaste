# Jazzmodem

재즈 피아노 화성과 리듬에 텍스트 데이터를 실어 보내는 브라우저 기반 음향 모뎀입니다.

네트워크, 서버, 계정, 설치 없이 두 기기의 스피커와 마이크만으로 짧은 텍스트를 전송하는 것을 목표로 합니다.

## Links

- Web: [jazzmodem.midagedev.com](https://jazzmodem.midagedev.com)
- Repository: [github.com/midagedev/jazzmodem](https://github.com/midagedev/jazzmodem)

## 현재 상태

Jazzmodem은 아직 실험 단계입니다. 현재 MVP는 12마디 재즈 블루스 진행 위에 화음 성부별로 데이터를 싣는 음악 모뎀을 중심으로 구현되어 있습니다.

- 송신/수신 단일 페이지
- 재즈 피아노풍 합성음 기반 기호 전송
- 138BPM 스윙 8분 그리드와 12마디 C 재즈 블루스 코드 진행
- 화음 성부 독립 변조: 베이스·가이드 톤이 코드를 짚고 그 위 3개 성부에 각 3비트(심볼당 9비트)
- 성부별 음역대 대역 탐색 복조로 저음량/잡음/리버브 내성 강화
- 스트리밍 표시를 위한 UTF-8 원문 청크 전송
- CRC-16/CCITT 기반 청크 검증
- GF(256) Reed-Solomon 패리티 청크로 소실된 청크 정정(FEC)
- 수신 중 CRC가 통과한 연속 청크를 textarea에 즉시 반영
- 전송 시간, 텍스트 크기, 프레임 크기 미리보기
- 송신 중 VU 미터와 코드 톤 EQ 시각화
- 브라우저 로케일 기반 한국어/영어 UI
- Node 내장 테스트 러너 기반 변복조 테스트

## 전송 컨셉

Jazzmodem은 단순 비프음 대신 음악적으로 들리는 전송음을 목표로 합니다. 현재 구현은 138BPM 스윙 8분 그리드와 12마디 C 재즈 블루스 진행을 따라갑니다. 각 화음은 코드 정체성을 짚는 고정 성부(베이스 + 3·7도 가이드 톤)와 데이터를 싣는 3개 성부로 이루어지며, 각 데이터 성부는 자기 음역대의 코드 스케일 음 8개 중 하나를 골라 3비트씩, 심볼당 9비트를 보냅니다. 컴핑 리듬은 데이터가 아니라 기호 순서로 정해지는 음악 질감입니다.

데이터는 화음 성부 자체에 실리므로, 수신기는 성부별 음역대를 괴르첼로 따로 읽어 후보 주파수 주변 대역에서 피크를 잡습니다. 전체 음량의 크기에 의존하지 않아 저음량·대역제한·리버브 상황에서 덜 흔들리고, 모든 화음이 슬롯 내내 지속되므로 마이크-스피커 음향 채널의 반향과 타이밍 오차에도 강합니다.

오류 처리는 CRC-16/CCITT로 깨진 청크를 검출하고, GF(256) Reed-Solomon erasure 코드로 정정합니다. 본문을 64B 데이터 청크로 나누고 블록마다 패리티 청크를 더해 보내며(블록당 패리티 수 ≈ 데이터 청크 수의 1/4, 최소 2), CRC가 어느 청크가 깨졌는지 알려주므로 라그랑주 보간만으로 임의의 소실 청크를 복원합니다. 단순 반복과 달리 버스트 오류에 강하고, 살아남은 청크 수가 데이터 청크 수 이상이면 전체를 되살립니다.

## 원칙

- No network: CSP에서 `connect-src 'none'`
- No backend: 정적 파일만 배포
- No runtime dependencies: 앱 런타임은 브라우저 API만 사용
- No payload storage: 전송 텍스트는 메모리에서만 처리

## 개발

```bash
npm install
npm run dev
```

## 테스트

```bash
npm test
```

현재 테스트는 CRC, 음악 기호 왕복, Reed-Solomon 패리티 복원과 한도 초과 실패, 부분 프레임 읽기, 저음량+잡음 조건의 합성 오디오 복조, 리버브와 타이밍 오프셋이 섞인 채널에서의 텍스트 복구, 짧은 텍스트 전체 왕복을 검증합니다.

## 빌드

```bash
npm run build
```

정적 산출물은 `dist/`에 생성됩니다.

Cloudflare Pages용 빌드는 커스텀 도메인 `jazzmodem.midagedev.com`에 맞춰 `/` base path를 사용합니다.

```bash
npm run build:cf
```

## 브라우저 요구사항

송신은 Web Audio API를 사용합니다. 수신은 마이크 권한이 필요하므로 배포 환경에서는 HTTPS가 필요합니다.

본문은 deflate(`CompressionStream`)로 압축한 뒤 청크로 나누어 전송하며, 압축 결과가 원문보다 크면 자동으로 비압축으로 되돌립니다.
